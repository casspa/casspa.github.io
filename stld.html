STAR TREK LOWER DECKS: THE BADGEY DIRECTIVE

UI Designer | MIGHTY KINGDOM

Figma | Unity | Sourcetree | Github | Atlassian | Miro

Joining the team shortly before world wide release, and working closely with multiple disciplines to create and implement numerous live and upcoming features in Unity, and identify and find solutions to possible UI/UX improvements

I also had regular communication with clients to ensure our designs met their best practices and the IP branding

Becoming part of this team around the time of the soft release, I had the opportunity to work on this Star Trek IdleGame, based on the Lower Decks shows

My initial focus was prototyping solutions for UX fixes, before the game was shipped, including creating clearer language and visuals for viewing and upgrading characters, ensuring all panels had consistent layouts for closing or accepting, adding clearer notifications on the HUD to show new/available upgrades, and accessibility testing to ensure all colours passed a colour contrast checker



Working with Game Designers, Tech Art, Programmers, QA and clients, I also designed and helped to implement multiple features such as the Filtering and Sorting of the Character Screen, Cloud Saves and the Credits Panel



During my time on this project, I had a great opportunity to work with the technical team and learn how to implement UI in Unity, as well as UI bug fixing. While there was a lot to learn, I found this to be an invaluable experience and an understanding of how UI is implemented also helped to influence future designs.

For my tasks I ensured that I added documentation, mockups and screenshots for anyone who would need to reference my work, such as programmers adding functionality or when it was ready for QA to test.

For consistency, I created a responsive panel prefab, for other UI designers or programmers, to use as a template for future panels. This contained almost any elements we would need (header, text, image, scrollview) in a layout group that would adapt, based on the size of the content
